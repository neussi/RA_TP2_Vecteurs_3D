\section{Introduction}

Ce rapport presente le module \textbf{Vecteurs 2D et 3D} du cours \textit{Realite Augmentee FROM SCRATCH}.
Les vecteurs sont la base geometrique de tout pipeline 3D : positions, directions, normales.

\textbf{Contrainte absolue :} aucune bibliotheque tierce, uniquement C++17 et STL.

\section{Concepts Theoriques}

\subsection{Vec2d -- Vecteur 2D}

Structure de 16 octets (2 doubles) avec les operations fondamentales :
\[
    \vec{u} \cdot \vec{v} = u_x v_x + u_y v_y \qquad
    \text{Cross2D}(\vec{u},\vec{v}) = u_x v_y - u_y v_x
\]

\subsection{Vec3d -- Vecteur 3D}

Le produit vectoriel en 3D :
\[
    \vec{u} \times \vec{v} = \begin{vmatrix} \vec{i} & \vec{j} & \vec{k} \\ u_x & u_y & u_z \\ v_x & v_y & v_z \end{vmatrix}
\]

La normalisation : $\hat{u} = \dfrac{\vec{u}}{||\vec{u}||}$, avec $||\vec{u}|| = \sqrt{u_x^2 + u_y^2 + u_z^2}$

\subsection{Algorithme de Gram-Schmidt}

Orthogonalisation d'une base quelconque :
\[
    \vec{u}_1 = \hat{a}, \quad \vec{u}_2 = \widehat{b - \text{proj}_{\vec{u}_1} b}, \quad \vec{u}_3 = \widehat{c - \text{proj}_{\vec{u}_1} c - \text{proj}_{\vec{u}_2} c}
\]

\section{Realisation Technique}

\subsection{Fichier Vec.h}

\begin{lstlisting}[language=C++, caption=Structure Vec2d]
struct Vec2d {
    double x, y;
    Vec2d(double x = 0, double y = 0) : x(x), y(y) {}
    double Norm()  const { return std::sqrt(x*x + y*y); }
    Vec2d Normalized() const { double n = Norm(); return {x/n, y/n}; }
    double& operator[](int i) { return i == 0 ? x : y; }
};
inline double Dot(Vec2d a, Vec2d b) { return a.x*b.x + a.y*b.y; }
inline double Cross2D(Vec2d a, Vec2d b) { return a.x*b.y - a.y*b.x; }
\end{lstlisting}
